Repo structure: `backend/` is the FastAPI + LangGraph agent itself (`agent.py`, `retrieval.py`, `citations.py`, prompts, `tests/`); `evals/` holds the golden set, the three-stage harness (`run_baseline.py` generation, `score.py` Ragas + rubric scoring, `compile_report.py` report compilation, `compare_runs.py` diffing), and every dated report/comparison under `evals/results/`; `frontend/` is the Next.js app deployed to Vercel; `docs/` holds `submission.md` (the written deliverable), this file, and `findings-log.md` (per-row eval detail). To reproduce a number, run `uv run run_baseline.py --out results/<name>.jsonl` from `evals/` against a local LiteLLM gateway with the repo-root `.env` credentials loaded, then `uv run score.py` and `uv run compile_report.py` on that same output file.
